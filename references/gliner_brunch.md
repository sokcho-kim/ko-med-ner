## 개요

**NER(Named Entity Recognition, 개체명 인식)**은 "텍스트문장에서 개체를 찾아서 미리 정의된 범주(사람 이름, 조직, 위치, 시간, 날짜 등)으로 분류하는 자연어 처리(NLP) 방법"입니다.

## 엔티티(Entity)란

예시: "수요일 부산 날씨 어떨까?"라는 질문에서:
- **DATE**: 수요일
- **LOC(위치)**: 부산

챗봇은 사용자의 의도를 파악한 후 정확한 시간과 위치 정보를 추출하여 적절한 응답을 제공할 수 있습니다.

## NER이 필요한 이유

### 1. 의미 모호성 해결

"아마존이 신제품을 출시했다"에서 아마존은:
- 지역명 또는
- 회사명/조직명

NER을 통해 문맥에서 정확한 의미를 식별할 수 있어 번역 품질과 사용자 경험이 향상됩니다.

### 2. 실제 활용

- 챗봇 질의응답
- 맞춤형 번역
- AI 채용 및 이력서 심사
- 후보자 자동 매칭

## LLM 시대의 NER

| 시대 | 방법 |
|------|------|
| 과거 | BiLSTM-CRF 같은 딥러닝 기술 필요 |
| 현재 | ChatGPT, GPT-4, LLaMA3 등 대규모 생성 언어모델으로 쉽게 구현 가능 |

## GLiNER 소개

**GLiNER**는 "BERT(BiDirectional Encoder Representation of Transformer) 기술을 사용하는 NER 모델"로, 모든 유형의 엔터티를 식별하도록 훈련된 소형 모델입니다.

### 특징

| 항목 | 내용 |
|------|------|
| 모델 | urchade/gliner_multi |
| 크기 | 약 1.33GB |
| 실행 환경 | 로컬환경에서 실행 가능 |
| 공식 지원 언어 | 영어, 프랑스어, 독일어, 스페인어, 이탈리아어, 포르투갈어 |
| 한국어 | 비공식적으로 잘 작동 |

### 테스트 결과

영어와 한국어 모두 엔터티 추출이 정상적으로 작동함을 확인했습니다.
